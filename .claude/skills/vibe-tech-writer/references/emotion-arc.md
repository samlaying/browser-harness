# 情绪弧线设计指南

## 情绪类型库

| 情绪 | 触发方式 | 适用位置 |
|------|---------|---------|
| 好奇 | 用一个反常识的事实或问题 | 开头 |
| 惊讶 | 用一个数据或转折 | 开头/中段 |
| 恐惧 | 用"你的XX正在被XX"的句式 | 开头/中段 |
| 恍然大悟 | 用历史溯源或"原来如此"的揭秘 | 中段 |
| 实用感 | 用具体操作步骤或工具介绍 | 中段 |
| 兴奋 | 用新能力、新发现 | 中段 |
| 警觉 | 用负面案例或边界模糊的灰色地带 | 中后段 |
| 震撼 | 用大厂动作或行业级事件 | 中后段 |
| 深思 | 用一个有力量的比喻或判断收束 | 结尾 |
| 信服 | 用硬数据支撑 | 中段/结尾 |
| 不安 | 用未知的规则或无人能答的问题 | 后段 |
| 释然 | 先打后扶，告诉读者"这不是你的错" | 中段 |
| 力量感 | 告诉读者"你可以做第二个了" | 结尾 |

## 情绪弧线模板

### 模板1：恐惧→理解→升维（适合技术趋势类）
```
恐惧 → 好奇 → 理解 → 恍然大悟 → 深思
```
来源：Codex Record & Replay

### 模板2：好奇→翻转→警觉→震撼→深思（适合行业观察类）
```
好奇 → 恍然大悟 → 惊讶 → 警觉 → 震撼 → 深思
```
来源：灵动岛

### 模板3：惊喜→实用→理性→感慨（适合功能报道类）
```
惊喜 → 实用感 → 兴奋 → 理性 → 感慨
```
来源：Claude Code Artifacts

### 模板4：震惊→好奇→理解→信服→深思（适合商业分析类）
```
震惊 → 好奇 → 理解 → 信服 → 深思
```
来源：演语科技融资

### 模板5：共情→释然→认知冲击→力量感（适合观点/社区洞察类）
```
共情/扎心 → 释然 → 认知冲击 → 实用感 → 力量感
```
来源：手帐文章

## 设计规则

1. 开头情绪必须是"好奇"或"惊讶"或"恐惧"或"共情"
2. 结尾情绪必须是"深思"或"恍然大悟"或"力量感"
3. 相邻两个情绪点不能相同
4. 全文至少有4个情绪转折点
5. 中段必须有一个"认知冲击"——给读者一个"原来如此"的时刻
6. 三条以上建议/清单的部分不能变成平坦的"实用感"，必须在其中埋至少一个情绪小高潮
