# 5模块内容结构详解

## 总体骨架

```
[钩子层 10%] → [背景层 20%] → [核心层 40%] → [对比层 15%] → [升维层 15%]
```

---

## 模块1：钩子层（占全文10%）

**目标**：3句话拽人入戏，建立"这件事跟你有关"的紧迫感

**技法**：参见 [opening-patterns.md](opening-patterns.md)

**检查标准**：
- 读完开头，读者必须产生"然后呢？"的冲动
- 至少有2种五感还原
- 有明确的时间或空间锚点

---

## 模块2：背景层（占全文20%）

**目标**：建立坐标系，让读者知道"这件事从哪来"

**核心技法**：
- 用"从XX到XX"的叙事线
- 有时间线或历史对比
- 找到一个"被忽略的事实"作为认知支点

**示例**（灵动岛文章）：
> 灵动岛一开始是苹果为了掩饰刘海屏的设计缺陷，以技术创新实现的一个补救方案。……苹果的解法很巧，干脆把这块挖孔区域做成一道会伸缩变形的黑色横条。

**检查标准**：
- 读者读完背景层后，应该能回答"这是什么"和"为什么是现在"
- 背景层不能超过全文的25%

---

## 模块3：核心层（占全文40%）

**目标**：文章的真正干货，拆解事件本身

**结构规则**：
- 每个小节用一个**判断句**做标题，不是模糊描述
- 小节之间用以下方式推进，绝不使用"首先/其次/最后"：
  - "但很快"（转折）
  - "不只XX"（扩展）
  - "更关键的是"（递进）
  - "还有一个"（补充）
  - "相比之下"（对比）

**每个小节的内容要求**：
- 必须有具体案例、数据或操作细节
- 必须有至少一个五感还原点
- 必须有至少一个口语化表达

**示例标题**：
- ✓ "你犯的错，不是做手帐——是把练习当成了产品"（判断句）
- ✓ "全员注意，AI 上岛！"（动作句）
- ✗ "灵动岛的发展历程"（模糊描述）
- ✗ "Claude Code 的新功能介绍"（模糊描述）

---

## 模块4：对比层（占全文15%）

**目标**：跟同类产品/方案/思路做对比，帮读者建立判断框架

**技法**：
- 不是为了比高低，是为了展示"路线差异"
- 用"XX更接近YY，XX更接近ZZ"的结构
- 列出具体的技术限制或设计选择，让读者自己判断

**示例**（Claude Code文章）：
> OpenAI 更接近应用平台，Anthropic 更接近工作画布。

**检查标准**：
- 对比不是简单的功能列表
- 必须有"设计哲学"层面的差异
- 不做最终判断，留给读者

---

## 模块5：升维层（占全文15%）

**目标**：把具体案例拉到行业/人类层面，给读者一个"转发理由"

**技法**：
- 最后一段必须是一个有力量的判断或比喻，不是总结
- 这句话要值得截图转发
- 可以用"回头看"开头，也可以直接下判断

**示例**：
> "一道当年差点被骂下去的刘海，如今成了兵家必争之地。它够小，小到只有一寸，可它的位置，恰好卡在我们和手机之间，最近的那道门口。"

**检查标准**：
- 结尾句不超过40字（可以是两句话）
- 读完会想截图发朋友圈
- 不使用"总的来说""简而言之"做收束
- 结尾不是重复前面说过的话，而是一个新的判断
